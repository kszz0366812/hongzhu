# 全局登录功能测试说明

## 测试步骤

### 1. 启动微信开发者工具
1. 打开微信开发者工具
2. 导入项目：`F:\minHzRcms`
3. 确保项目配置正确

### 2. 配置开发环境
1. 在微信开发者工具中，点击右上角的"详情"
2. 勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"
3. 确保后端服务器运行在 `localhost:8080`

### 3. 测试全局静默登录

#### 方法一：自动测试（推荐）
1. 在微信开发者工具中，点击"编译"
2. 应用启动后，会自动跳转到个人中心页面
3. 在底部导航栏中，点击"我的"
4. 在个人中心页面，点击"测试登录"按钮
5. 页面会自动执行全局静默登录测试
6. 查看页面上的日志输出，确认登录状态

#### 方法二：手动测试
1. 在微信开发者工具中，点击"编译"
2. 应用启动后，会自动执行全局静默登录
3. 查看控制台日志，确认是否有以下输出：
   ```
   App launched
   开始全局静默登录...
   wx.login成功: {code: "xxx"}
   获取到登录code: xxx
   API登录响应: {...}
   登录成功，数据已保存到本地存储
   全局静默登录成功: {...}
   ```

### 4. 测试新用户绑定流程

#### 模拟新用户
1. 清除本地存储：
   - 在微信开发者工具中，点击"Storage"面板
   - 删除所有存储项
2. 重新编译应用
3. 应用启动后，应该自动显示员工绑定弹窗
4. 输入员工号（如：EMP001）
5. 确认绑定成功

#### 验证绑定结果
1. 查看控制台日志，确认绑定成功
2. 检查本地存储，确认用户信息已更新
3. 查看页面状态，确认员工号已显示

### 5. 测试老用户登录

#### 模拟老用户
1. 确保本地存储中有有效的用户信息
2. 重新编译应用
3. 应用启动后，应该直接登录成功，不显示绑定弹窗
4. 查看控制台日志，确认登录流程

## 调试方法

### 1. 查看控制台日志
在微信开发者工具的控制台中，查看详细的日志输出：
```
App launched
开始全局静默登录...
wx.login成功: {code: "xxx"}
API登录响应: {...}
登录成功，数据已保存到本地存储
全局静默登录成功: {...}
```

### 2. 查看网络请求
在微信开发者工具的"Network"面板中，查看API请求：
- 请求URL：`http://localhost:8080/api/auth/wx/login`
- 请求方法：POST
- 请求数据：`{"code": "xxx"}`

### 3. 查看本地存储
在微信开发者工具的"Storage"面板中，查看存储的数据：
- `token`: JWT令牌
- `userInfo`: 用户信息
- `openid`: 微信OpenID
- `unionid`: 微信UnionID

### 4. 使用测试页面
访问测试登录页面，查看详细的测试日志：
1. 点击底部导航栏的"我的"
2. 点击"测试登录"
3. 查看页面上的操作日志

## 常见问题

### Q: 为什么没有自动执行登录？
A: 检查以下几点：
1. 确保 `app.js` 中的 `onLaunch` 方法正确调用了 `performGlobalSilentLogin()`
2. 确保微信开发者工具中勾选了"不校验合法域名"
3. 确保后端服务器正在运行

### Q: 为什么登录失败？
A: 检查以下几点：
1. 确保后端服务器运行在 `localhost:8080`
2. 确保API接口 `/auth/wx/login` 正常工作
3. 检查网络连接是否正常
4. 查看控制台错误信息

### Q: 为什么没有显示员工绑定弹窗？
A: 检查以下几点：
1. 确保后端返回的 `isNewUser` 字段为 `true`
2. 确保 `app.js` 中的 `showGlobalEmployeeBindingModal()` 方法正确实现
3. 检查是否有JavaScript错误

### Q: 如何清除登录状态？
A: 在微信开发者工具中：
1. 点击"Storage"面板
2. 删除所有存储项
3. 重新编译应用
