# 任务表单页面合并完成说明

## 概述

已完成任务创建页面和编辑页面的合并工作，使用操作码区分不同的操作模式，并删除了冗余的编辑页面代码。现在使用统一的`task-form`页面处理任务的创建和编辑操作。

## 主要变更

### 1. 页面合并

- **原页面**: `pages/create-task/` (创建任务页面)
- **新页面**: `pages/task-form/` (统一的任务表单页面)
- **删除页面**: `pages/edit-task/` (编辑任务页面，已删除)

### 2. 操作模式区分

使用`operationMode`字段区分操作模式：

```javascript
data: {
  // 操作模式：create-创建，edit-编辑
  operationMode: 'create',
  editingTaskId: null,
  // ... 其他数据
}
```

### 3. 动态页面标题

根据操作模式动态设置页面标题：

- **创建模式**: "创建任务"
- **编辑模式**: "编辑任务"

### 4. 动态按钮文本

根据操作模式动态显示按钮文本：

- **创建模式**: "创建任务"
- **编辑模式**: "更新任务"

## 功能特性

### 1. 创建任务模式

- 设置默认截止时间（7天后）
- 隐藏任务状态选择字段
- 调用`api.createTask()`接口
- 显示"创建任务"按钮

### 2. 编辑任务模式

- 加载现有任务数据
- 显示任务状态选择字段
- 调用`api.updateTask()`接口
- 显示"更新任务"按钮
- 支持状态更新

### 3. 通用功能

- 项目列表动态加载
- 员工列表动态加载
- 表单验证
- 附件上传
- 错误处理

### 2. 编辑任务数据设置

```javascript
setupEditingTask: function(task) {
  setTimeout(() => {
    // 找到对应的索引并设置表单数据
    const projectIndex = this.data.projects.findIndex(p => p.name === task.project);
    const assigneeIndex = this.data.teamMembers.findIndex(m => m.name === task.assignee);
    const statusIndex = this.data.taskStatuses.findIndex(s => s === task.status);
    const priorityIndex = this.data.priorities.findIndex(p => p === task.priorityText);
    
    this.setData({
      'taskForm.title': task.title,
      'taskForm.projectIndex': projectIndex !== -1 ? projectIndex : undefined,
      // ... 其他字段
      editingTaskId: task.id
    });
  }, 100);
}
```
