# 员工列表接口集成完成说明

## 概述
已成功将员工列表后台接口集成到微信小程序中，实现了邀请人员、指派人员和选择人员功能的统一数据源。

## 完成的修改

### 1. API配置更新 (`utils/api.js`)
- 添加了 `getEmployeeList()` 方法，调用后台员工列表接口 `/employee/list`
- 接口返回的数据格式会被转换为前端所需的格式
- 支持错误处理和重试机制

### 2. 项目负责人选择控件优化

#### 创建项目页面 (`pages/create-project/`)
- **WXML**: 将项目负责人选择从原来的picker改为自定义控件，样式参考任务指派控件
- **JS**: 
  - 添加了 `loadEmployeeList()` 方法在页面加载时获取员工数据
  - 添加了 `selectManager()` 打开负责人选择弹窗
  - 添加了 `onManagerSelect()` 和 `onManagerModalClose()` 处理负责人选择事件
  - 增加了 `showManagerModal` 和 `selectedManager` 数据字段
- **WXSS**: 添加了负责人选择器的自定义样式，包括头像、姓名、部门等信息展示

#### 编辑项目页面 (`pages/edit-project/`)
- **WXML**: 添加了项目负责人选择控件（之前没有）
- **JS**: 
  - 添加了 `loadEmployeeList()` 方法
  - 添加了完整的负责人选择功能和成员邀请功能
  - 增加了相应的数据字段和事件处理方法
- **WXSS**: 添加了与创建项目页面相同的负责人选择器样式

### 3. 任务指派页面更新

#### 指派任务页面 (`pages/assign-task/`)
- **JS**: 
  - 添加了 `loadEmployeeList()` 方法替换硬编码的团队成员数据
  - 优化了页面加载逻辑，确保员工数据加载完成后再设置任务数据

#### 创建任务页面 (`pages/create-task/`)
- **JS**: 添加了 `loadEmployeeList()` 方法，从API获取员工数据

#### 编辑任务页面 (`pages/edit-task/`)
- **JS**: 
  - 添加了 `loadEmployeeList()` 方法
  - 优化了编辑逻辑，确保员工数据加载后再设置表单数据

### 4. 数据格式转换
在所有页面中，API返回的员工数据都会被转换为统一的前端格式：
```javascript
{
  id: employee.id,
  name: employee.name || employee.employeeName || '未知',
  department: employee.department || '未知部门',
  role: employee.position || employee.role || '员工'
}
```

### 5. 成员选择器组件 (`components/member-selector/`)
- 组件本身保持不变，仍然是纯展示组件
- 数据由父组件通过API获取后传入
- 支持单选（负责人选择）和多选（成员邀请）模式

## 用户交互流程

### 项目负责人选择
1. 用户点击"请选择项目负责人"区域
2. 打开成员选择器弹窗，显示从API获取的员工列表
3. 用户可搜索和选择负责人
4. 选择后在表单中显示负责人的头像、姓名和部门信息

### 成员邀请
1. 用户点击"邀请成员"按钮
2. 打开成员选择器弹窗，显示可邀请的员工（排除已邀请的）
3. 用户可多选成员
4. 确认后添加到项目成员列表

### 任务指派
1. 在创建/编辑任务时，负责人选择使用picker控件
2. 数据来源从API获取的员工列表
3. 支持搜索和选择功能

## 错误处理
- 所有API调用都包含错误处理
- 401错误会触发重新登录流程
- 其他错误会显示友好的错误提示
- 如果API调用失败，会保留默认的员工数据确保页面正常使用

## 技术特点
- 统一的数据源：所有人员选择功能都使用同一个API接口
- 良好的用户体验：加载状态提示、错误处理、成功反馈
- 灵活的数据格式：兼容不同的后台数据字段名称
- 组件化设计：成员选择器可复用于不同场景
- 响应式设计：自定义控件具有良好的交互反馈
