# 服务端接口说明

## 基础信息
- 服务端地址：`http://localhost:8080/api`
- 请求格式：JSON（登录接口）或表单参数（绑定接口）
- 响应格式：JSON

## 接口列表

### 1. 微信登录接口
**接口地址：** `POST /auth/wx/login`

**请求参数：**
```json
{
  "code": "微信登录code",
  "userInfo": {
    "nickName": "用户昵称",
    "avatarUrl": "头像URL",
    "gender": 1,
    "country": "中国",
    "province": "广东",
    "city": "深圳"
  }
}
```

**响应格式：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "success": true,
    "token": "JWT令牌",
    "openid": "用户openid",
    "unionid": "用户unionid",
    "isNewUser": true,
    "userInfo": {
      "id": 1,
      "nickname": "用户昵称",
      "avatarUrl": "头像URL",
      "gender": 1,
      "country": "中国",
      "province": "广东",
      "city": "深圳",
      "employeeCode": null
    }
  }
}
```

**重要说明：**
- `isNewUser`: 布尔值，根据 `user.getEmployeeCode()==null` 判断
  - `true`: 新用户，`employeeCode` 为 `null`，需要绑定员工号
  - `false`: 老用户，`employeeCode` 不为 `null`，已绑定员工号
- 当 `isNewUser` 为 `true` 时，前端会显示两步员工绑定弹窗（无法取消）

### 2. 绑定员工信息接口
**接口地址：** `POST /auth/wx/bind-employee`

**请求头：**
```
Authorization: Bearer <token>
Content-Type: application/x-www-form-urlencoded
```

**请求参数：**
```
employeeCode=EMP001
```

**响应格式：**
```json
{
  "code": 200,
  "message": "绑定成功",
  "data": {
    "success": true,
    "employeeCode": "EMP001",
    "userInfo": {
      "id": 1,
      "nickname": "用户昵称",
      "avatarUrl": "头像URL",
      "gender": 1,
      "country": "中国",
      "province": "广东",
      "city": "深圳",
      "employeeCode": "EMP001"
    }
  }
}
```

**重要说明：**
- 此接口需要 JWT 鉴权，请在请求头中携带 `Authorization: Bearer <token>`
- 请求格式为表单参数（`application/x-www-form-urlencoded`），不是JSON
- 后端使用 `@RequestParam String employeeCode` 接收参数
- 绑定成功后需要返回更新后的用户信息（包含 `employeeCode`）
- 前端会用返回的用户信息替换本地存储的用户信息
- 新用户登录后会显示两步员工绑定弹窗进行绑定
- 只需要传递员工号，不需要手机号码

## 错误响应格式
```json
{
  "code": 400,
  "message": "错误信息",
  "data": null
}
```

## 常见错误码
- `400`: 请求参数错误
- `401`: 未授权（token无效）
- `404`: 接口不存在
- `500`: 服务器内部错误

## 静默登录流程

### 1. 自动登录
1. 小程序启动时自动执行 `wx.login()` 获取临时code
2. 调用后端 `/auth/wx/login` 接口进行登录
3. 服务端返回 `isNewUser` 状态和用户信息
4. 前端自动保存 token 和用户信息到本地存储

### 2. 新用户强制绑定流程
1. 用户首次登录时，服务端返回 `isNewUser: true`（基于 `employeeCode` 是否为 `null`）
2. 前端检测到新用户，显示两步员工绑定弹窗（无法取消）
3. 用户在弹窗中输入员工号完成绑定

### 3. 员工绑定流程（两步弹窗）
1. **第一步：确认弹窗**
   - 显示"员工号绑定"确认弹窗
   - 内容：检测到您是首次登录，必须绑定员工号才能使用系统功能。
   - 按钮：立即绑定（无取消按钮）

2. **第二步：输入弹窗**
   - 显示"输入员工号"输入弹窗
   - 内容：
   - 输入框：可编辑的员工号输入框
   - 按钮：确认绑定（无取消按钮）

3. **验证和绑定**
   - 验证员工号是否为空
   - 调用 `/auth/wx/bind-employee` 进行绑定
   - 绑定成功后显示成功提示
   - 绑定失败或员工号为空时，自动重新显示输入弹窗

### 4. 绑定完成
1. 用户完成员工号绑定
2. 服务端更新用户的 `employeeCode` 字段
3. 服务端返回更新后的用户信息
4. 前端用返回的用户信息替换本地存储的用户信息
5. 下次登录时 `isNewUser` 将为 `false`

## 服务端实现要点

### 1. 登录接口逻辑
```java
// 判断是否为新用户
boolean isNewUser = user.getEmployeeCode() == null;

// 返回结果
result.put("success", true);
result.put("token", token);
result.put("openid", openid);
result.put("unionid", unionid);
result.put("isNewUser", isNewUser);
result.put("userInfo", buildUserInfo(user));
```

### 2. 绑定接口逻辑
```java
// 绑定员工信息后，更新用户的 employeeCode
user.setEmployeeCode(employeeCode);
userService.updateUser(user);

// 返回绑定成功结果，包含更新后的用户信息
result.put("success", true);
result.put("employeeCode", employeeCode);
result.put("userInfo", buildUserInfo(user)); // 返回更新后的用户信息
```

## 静默登录特性

### 1. 自动登录
- 小程序启动时自动执行登录流程
- 无需用户手动点击登录按钮
- 登录状态自动保存到本地存储

### 2. 无退出登录
- 用户无需手动退出登录
- 系统自动管理登录状态
- 登录信息持久化存储

### 3. 自动重连
- 网络断开后自动重连
- 登录状态失效时自动重新登录
- 保证用户始终处于登录状态

### 4. 强制绑定
- 新用户必须绑定员工号才能使用系统功能
- 绑定弹窗无法取消，必须完成绑定
- 使用两步弹窗流程，提供更好的用户体验
- 绑定失败或员工号为空时自动重新显示输入弹窗

## 注意事项
1. 所有接口都需要返回统一的响应格式
2. 微信登录接口需要调用微信的 `code2Session` 接口获取 openid
3. 建议使用 JWT 作为身份验证令牌
4. 绑定员工信息接口需要 JWT 鉴权
5. 新用户登录后会显示两步员工绑定弹窗（无法取消）
6. `isNewUser` 字段基于 `employeeCode` 是否为 `null` 来判断
7. 绑定接口只需要员工号，不需要手机号码
8. 使用两步弹窗形式，提供更好的输入体验
9. 静默登录无需用户手动操作，系统自动管理登录状态
10. 无需提供退出登录功能，登录状态由系统自动维护
11. 员工号绑定是强制性的，用户无法跳过或取消
12. 绑定失败时会自动重新显示输入弹窗，确保用户完成绑定
13. 两步弹窗流程解决了输入框样式问题，提供更好的用户体验
14. 员工号验证只检查是否为空，不进行格式验证
15. 接口参数统一使用 `employeeCode` 作为员工号字段名
16. 绑定接口使用表单参数格式（`application/x-www-form-urlencoded`），不是JSON格式
17. 绑定成功后需要返回更新后的用户信息，前端会用其替换本地存储的用户信息
