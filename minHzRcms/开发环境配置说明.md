# 开发环境配置说明

## 域名校验问题解决方案

### 问题描述
微信小程序要求所有网络请求的域名必须在微信公众平台后台配置的合法域名列表中，否则会出现以下错误：
```
https://your-domain.com 不在以下 request 合法域名列表中
```

### 解决方案

#### 1. 开发环境临时解决方案

**方法一：关闭域名校验（推荐）**
1. 在微信开发者工具中，点击右上角的"详情"
2. 在"本地设置"中，勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"
3. 重新编译项目

**方法二：使用项目配置**
项目已配置 `project.config.json` 中的 `urlCheck: false`，但建议在开发者工具中手动确认设置。

#### 2. 使用模拟API（已实现）

项目已实现模拟API功能，在开发环境中会自动使用模拟数据：

- **模拟登录**: 返回模拟的 token、openid、用户信息
- **模拟用户信息**: 返回测试用户数据
- **模拟员工绑定**: 支持测试员工绑定功能

#### 3. 配置本地服务器

如果需要连接真实后端API，可以：

1. **启动本地服务器**:
   ```bash
   # 示例：启动本地Node.js服务器
   npm install -g http-server
   http-server -p 8080
   ```

2. **配置API地址**:
   在 `config/wx-config.js` 中修改：
   ```javascript
   API_BASE_URL: 'http://localhost:8080/api'
   ```

#### 4. 生产环境配置

在生产环境中，需要：

1. **配置合法域名**:
   - 登录微信公众平台
   - 进入"开发"→"开发管理"→"开发设置"
   - 在"服务器域名"中添加你的API域名

2. **使用HTTPS**:
   - 生产环境必须使用HTTPS协议
   - 确保SSL证书有效

3. **更新配置**:
   ```javascript
   API_BASE_URL: 'https://your-production-domain.com/api'
   ```

## 测试步骤

1. **启动项目**:
   - 在微信开发者工具中打开项目
   - 确保已关闭域名校验

2. **测试登录**:
   - 小程序启动时会自动执行静默登录
   - 查看控制台输出和页面显示

3. **测试用户信息**:
   - 进入个人页面
   - 点击"获取用户信息"按钮

4. **测试员工绑定**:
   - 进入个人资料页面
   - 点击"绑定员工信息"
   - 输入手机号 `13800138000` 进行测试

## 注意事项

1. **开发环境**: 使用模拟API，无需真实后端服务
2. **生产环境**: 必须配置合法域名和HTTPS证书
3. **调试**: 查看控制台输出了解API调用情况
4. **测试**: 使用提供的测试数据进行功能验证

## 常见问题

**Q: 为什么还是出现域名错误？**
A: 请确保在微信开发者工具中已关闭域名校验，或使用模拟API功能。

**Q: 如何切换到真实API？**
A: 修改 `config/wx-config.js` 中的 `API_BASE_URL`，并确保域名已在微信公众平台配置。

**Q: 模拟API数据可以修改吗？**
A: 可以，编辑 `utils/mock-api.js` 文件修改模拟数据。
