# 全局登录功能使用说明

## 概述

本系统已实现真正的全局静默登录功能。**登录过程在应用启动时（`app.js`）自动执行**，无需在页面中手动处理。如果检测到新用户，系统会自动显示员工绑定弹窗，确保用户完成绑定后才能使用系统功能。

## 核心功能

### 1. 全局静默登录
- **应用启动时自动执行**：在 `app.js` 的 `onLaunch` 和 `onShow` 中自动执行
- **无需页面干预**：页面无需手动调用登录接口
- **自动重连机制**：网络断开后自动重新登录
- **状态自动管理**：登录状态自动保存到本地存储

### 2. 自动员工绑定
- **新用户检测**：登录成功后自动检测 `isNewUser` 状态
- **强制绑定流程**：新用户自动显示两步员工绑定弹窗
- **无法跳过**：绑定弹窗无法取消，必须完成绑定
- **自动更新**：绑定成功后自动更新用户信息

### 3. 全局权限控制
- **统一管理**：所有登录逻辑在 `app.js` 中统一处理
- **页面简化**：页面无需关心登录状态检查
- **自动跳转**：未登录用户自动引导到登录流程

## 工作原理

### 1. 应用启动流程
```javascript
// app.js onLaunch
onLaunch: function() {
  // 立即执行全局静默登录
  this.performGlobalSilentLogin()
}

// app.js onShow  
onShow: function() {
  // 检查登录状态，如果未登录则重新登录
  if (!auth.checkLoginStatus()) {
    this.performGlobalSilentLogin()
  }
}
```

### 2. 全局静默登录流程
```javascript
performGlobalSilentLogin: function() {
  // 1. 检查是否已登录
  if (auth.checkLoginStatus()) return
  
  // 2. 执行微信登录
  return auth.wxSilentLogin()
    .then((data) => {
      // 3. 更新全局数据
      this.updateGlobalData()
      
      // 4. 检查是否为新用户
      if (data.isNewUser === true) {
        // 5. 自动显示员工绑定弹窗
        this.showGlobalEmployeeBindingModal()
      }
    })
}
```

### 3. 员工绑定流程
```javascript
showGlobalEmployeeBindingModal: function() {
  // 第一步：确认弹窗
  wx.showModal({
    title: '员工号绑定',
    content: '检测到您是首次登录，必须绑定员工号才能使用系统功能。',
    showCancel: false,
    confirmText: '立即绑定',
    success: (modalRes) => {
      if (modalRes.confirm) {
        // 第二步：输入弹窗
        this.showGlobalEmployeeInputPrompt()
      }
    }
  })
}
```

## 使用方法

### 1. 创建新页面（简化版）

#### 方法一：使用页面基类（推荐）
```javascript
// pages/example/example.js
const { createPage } = require('../../utils/page-base.js')

createPage({
  data: {
    // 页面数据
  },

  // 页面加载时的处理
  pageOnLoad: function(options) {
    // 页面初始化逻辑
    // 无需关心登录状态，登录已在全局处理
  },

  // 页面显示时的处理
  pageOnShow: function() {
    // 页面显示逻辑
    // 无需关心登录状态，登录已在全局处理
  }
})
```

#### 方法二：使用权限装饰器
```javascript
// pages/example/example.js
const { requireLogin } = require('../../utils/login-manager.js')

Page(requireLogin({
  data: {
    // 页面数据
  },

  onLoad: function(options) {
    // 页面初始化逻辑
  }
}))
```

### 2. 页面跳转

#### 使用登录管理器进行跳转
```javascript
const { LoginManager } = require('../../utils/login-manager.js')

// 普通跳转
LoginManager.navigateWithCheck('/pages/example/example')

// 重定向
LoginManager.navigateWithCheck('/pages/example/example', 'redirectTo')

// 切换标签页
LoginManager.navigateWithCheck('/pages/example/example', 'switchTab')
```

#### 使用页面基类方法
```javascript
// 在页面中使用基类提供的方法
this.navigateTo('/pages/example/example')
this.redirectTo('/pages/example/example')
this.switchTab('/pages/example/example')
```

### 3. 获取用户信息

```javascript
const { LoginManager } = require('../../utils/login-manager.js')

// 获取用户信息
const userInfo = LoginManager.getUserInfo()

// 获取员工号
const employeeCode = LoginManager.getEmployeeCode()

// 获取用户昵称
const nickname = LoginManager.getNickname()

// 获取用户头像
const avatarUrl = LoginManager.getAvatarUrl()

// 检查登录状态
const isLoggedIn = LoginManager.isLoggedIn()

// 检查是否已绑定员工号
const hasBinding = LoginManager.hasEmployeeBinding()
```

### 4. 监听登录状态变化

```javascript
const { LoginManager } = require('../../utils/login-manager.js')

// 监听登录成功
LoginManager.onLoginStatusChange((data) => {
  console.log('登录成功:', data)
  // 更新页面数据
})

// 监听登录失败
LoginManager.onLoginFail((error) => {
  console.log('登录失败:', error)
  // 处理登录失败
})

// 监听绑定成功
LoginManager.onBindingSuccess((data) => {
  console.log('绑定成功:', data)
  // 处理绑定成功
})
```

- 处理网络异常情况

## 常见问题

### Q: 为什么页面无法访问？
A: 检查用户是否已登录并绑定员工号，使用测试页面查看登录状态。

### Q: 如何调试登录问题？
A: 使用 `pages/test-login/test-login` 页面进行调试，查看控制台日志。

### Q: 如何修改权限控制？
A: 修改 `utils/login-manager.js` 中的白名单配置和权限检查逻辑。

### Q: 如何添加新的API接口？
A: 在 `utils/api.js` 中添加新的接口函数，确保包含正确的认证头。

### Q: 如何自定义登录流程？
A: 修改 `app.js` 中的 `performGlobalSilentLogin` 方法，或使用 `LoginManager` 的回调函数。

### Q: 登录是在哪里处理的？
A: **登录在 `app.js` 中全局处理**，页面无需关心登录状态，系统会自动管理。
