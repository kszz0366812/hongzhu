# 任务表单编辑模式绑定修复说明

## 问题描述

在任务表单的编辑模式下，负责人和所属项目字段没有正确绑定到编辑的任务数据上，导致：
- 编辑页面打开后，负责人和所属项目显示为"请选择"
- 无法正确显示原有任务的项目和负责人信息
- 用户需要重新选择这些字段

## 问题原因分析

### 1. 数据加载时序问题

**问题现象：**
- `setupEditingTask` 在 `onLoad` 中被立即调用
- 但 `loadProjectList` 和 `loadEmployeeList` 是异步操作
- 当设置编辑任务数据时，项目列表和员工列表可能还未加载完成

**代码问题：**
```javascript
// 问题代码
onLoad: function(options) {
  if (options.task) {
    // 立即调用，但数据可能未加载
    this.setupEditingTask(task);
  }
  
  // 异步加载数据
  this.loadProjectList();
  this.loadEmployeeList();
}
```

### 2. 索引查找失败

**问题现象：**
- 在数据未加载完成时调用 `findIndex`
- 返回 -1，导致索引设置失败
- 最终显示为未选择状态

**代码问题：**
```javascript
// 问题代码
const projectIndex = this.data.projects.findIndex(p => p.name === task.project);
const assigneeIndex = this.data.teamMembers.findIndex(m => m.name === task.assignee);

// 当 projects 或 teamMembers 为空数组时，findIndex 返回 -1
```

### 3. 延迟处理不够

**问题现象：**
- 使用 `setTimeout` 延迟100ms
- 但网络请求可能超过100ms
- 延迟时间不够，导致数据仍未加载完成

## 修复方案

### 1. 调整数据设置时机

**修复思路：**
- 不在 `onLoad` 中立即设置编辑任务数据
- 将任务数据保存到 `editingTask` 字段
- 等待项目列表和员工列表加载完成后再设置

**修复代码：**
```javascript
onLoad: function(options) {
  if (options.task) {
    try {
      const task = JSON.parse(decodeURIComponent(options.task));
      // 保存任务数据，等数据加载完成后再设置
      this.setData({
        editingTask: task
      });
    } catch (error) {
      // 错误处理
    }
  }
  
  // 异步加载数据
  this.loadProjectList();
  this.loadEmployeeList();
}
```

### 2. 在数据加载完成后设置编辑任务

**修复思路：**
- 在 `loadProjectList` 和 `loadEmployeeList` 完成后
- 检查是否是编辑模式且有编辑任务数据
- 如果是，则调用 `setupEditingTask` 设置数据

**修复代码：**
```javascript
// 在项目列表加载完成后
this.setData({
  projects: projects
});

// 如果是编辑模式且项目列表已加载，尝试设置编辑任务数据
if (this.data.operationMode === 'edit' && this.data.editingTask) {
  this.setupEditingTask(this.data.editingTask);
}

// 在员工列表加载完成后
this.setData({
  teamMembers: teamMembers
});

// 如果是编辑模式且员工列表已加载，尝试设置编辑任务数据
if (this.data.operationMode === 'edit' && this.data.editingTask) {
  this.setupEditingTask(this.data.editingTask);
}
```

### 3. 优化 setupEditingTask 方法

**修复思路：**
- 移除 `setTimeout` 延迟处理
- 添加数据完整性检查
- 增加详细的日志输出，便于调试

**修复代码：**
```javascript
setupEditingTask: function(task) {
  console.log('设置编辑任务数据:', task);
  console.log('当前项目列表:', this.data.projects);
  console.log('当前员工列表:', this.data.teamMembers);
  
  // 确保数据已加载
  if (!this.data.projects || this.data.projects.length === 0 || 
      !this.data.teamMembers || this.data.teamMembers.length === 0) {
    console.log('数据未完全加载，跳过设置编辑任务数据');
    return;
  }
  
  // 找到对应的索引
  const projectIndex = this.data.projects.findIndex(p => p.name === task.project);
  const assigneeIndex = this.data.teamMembers.findIndex(m => m.name === task.assignee);
  
  console.log('找到的索引:', { projectIndex, assigneeIndex });
  
  // 设置表单数据
  this.setData({
    'taskForm.projectIndex': projectIndex !== -1 ? projectIndex : undefined,
    'taskForm.assigneeIndex': assigneeIndex !== -1 ? assigneeIndex : undefined,
    // ... 其他字段
  });
}
```

## 技术实现细节

### 1. 数据流控制

```
onLoad → 保存编辑任务数据 → 异步加载项目/员工列表 → 设置编辑任务数据
```

### 2. 状态管理

- `editingTask`: 存储待编辑的任务数据
- `operationMode`: 标识当前操作模式
- `projects`/`teamMembers`: 存储下拉选项数据

### 3. 错误处理

- 数据解析失败时的错误提示
- 网络请求失败时的降级处理
- 数据不完整时的跳过处理
