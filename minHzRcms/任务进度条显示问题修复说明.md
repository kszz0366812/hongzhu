# 任务进度条显示问题修复说明

## 问题描述

在任务管理页面中，任务卡片错误地显示了进度条，显示"进度 0%"。但实际上任务本身并没有进度功能，只有项目才有进度功能。

## 问题原因分析

### 1. 组件逻辑错误

在`list-item-card`组件中，存在错误的进度条显示逻辑：

```javascript
// 错误的逻辑（修复前）
// 只有任务才需要进度条，报告不需要
if (item.progress === undefined && item.priority !== '日报' && item.priority !== '周报') {
  item.progress = 0;
}
if (!item.progressLabel) item.progressLabel = '进度';
```

这个逻辑有两个问题：
- 注释说"只有任务才需要进度条"，但实际上是错误的
- 任务不应该有进度条，只有项目才有进度条

### 2. 任务类型标识不明确

任务页面中的任务类型设置不明确，导致组件无法正确区分任务和项目。

## 修复方案

### 1. 修复组件逻辑

更新`list-item-card`组件的进度条显示逻辑：

```javascript
// 修复后的逻辑
// 只有项目才需要进度条，任务和报告不需要
if (item.progress === undefined && item.type === 'project') {
  item.progress = 0;
}
if (!item.progressLabel && item.type === 'project') item.progressLabel = '进度';
```

### 2. 明确任务类型标识

在任务页面中，明确设置任务类型为`'task'`：

```javascript
// 确定任务类型
determineTaskType: function(task) {
  // 任务类型标识，用于区分任务和项目
  return 'task';
}
```

### 3. 确保任务数据不包含进度信息

在任务数据转换时，明确设置进度为`undefined`：

```javascript
convertTaskData: function(backendTasks) {
  return backendTasks.map(task => ({
    // ... 其他字段
    // 明确设置进度为undefined，避免显示进度条
    progress: undefined,
    progressLabel: undefined
  }));
}
```

### 4. 更新筛选逻辑

由于任务类型统一为`'task'`，更新筛选逻辑：

```javascript
// 按任务类型筛选（一级标签）
switch (activePrimaryTab) {
  case 'myTasks':
    // 所有任务都显示，因为现在统一使用task类型
    filteredTasks = [...tasks];
    break;
  case 'assignedByMe':
    // 所有任务都显示，因为现在统一使用task类型
    filteredTasks = [...tasks];
    break;
  case 'assignedToMe':
    // 所有任务都显示，因为现在统一使用task类型
    filteredTasks = [...tasks];
    break;
}
```

## 修复后的效果

- ✅ 只显示任务相关信息（标题、状态、优先级、项目、负责人、截止时间、描述）
- ✅ 保持原有的操作按钮（指派、编辑、开始任务、完成任务）

### 2. 项目卡片

- ✅ 继续显示进度条（如果有进度信息）
- ✅ 显示项目相关信息

### 3. 报告卡片

- ✅ 不显示进度条
- ✅ 显示报告相关信息

## 技术实现细节

### 1. 组件类型区分

通过`item.type`字段区分不同类型的卡片：

- `'task'` - 任务卡片，不显示进度条
- `'project'` - 项目卡片，显示进度条
- `'report'` - 报告卡片，不显示进度条

### 2. 数据转换

在任务数据转换时，确保不包含进度信息：

```javascript
// 明确设置进度相关字段为undefined
progress: undefined,
progressLabel: undefined
```

### 3. 组件逻辑优化

组件根据类型智能决定是否显示进度条：

```javascript
// 只有项目类型才设置默认进度
if (item.progress === undefined && item.type === 'project') {
  item.progress = 0;
}
```
