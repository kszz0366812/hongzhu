# 用户信息显示更新说明

## 更新内容

### 1. 个人中心页面 (`pages/personal/personal.wxml`)
- **修改前**: 显示 `{{userInfo.nickName || '微信用户'}}`
- **修改后**: 显示 `{{userInfo.realName || userInfo.nickName || '微信用户'}}`
- **效果**: 优先显示真实姓名，如果没有则显示昵称，最后显示默认值

### 2. 个人资料页面 (`pages/personal-info/personal-info.wxml`)
- **新增**: 添加"姓名"字段，显示 `{{userInfo.realName || '未设置'}}`
- **修改**: 头像下方名称优先显示真实姓名
- **修改**: 员工号字段从 `employeeId` 改为 `employeeCode`

### 3. 测试页面 (`pages/test-login/test-login.wxml`)
- **新增**: 显示真实姓名和员工号信息
- **新增**: 测试结果显示区域

### 4. 全局事件通知系统
- **新增**: `app.js` 中的 `updateGlobalData` 方法会通知所有页面更新用户信息
- **新增**: 各页面添加 `onUserInfoUpdate` 回调方法，实时接收用户信息更新

## 用户信息字段说明

### 后端返回的用户信息结构
```javascript
{
  "realName": "张三",        // 真实姓名（优先显示）
  "nickName": "微信昵称",    // 微信昵称（备用显示）
  "employeeCode": "EMP001", // 员工号
  "avatarUrl": "...",       // 头像URL
  "gender": 1,              // 性别（1=男，2=女）
  "country": "中国",        // 国家
  "province": "广东",       // 省份
  "city": "深圳"           // 城市
}
```

### 显示优先级
1. **真实姓名** (`realName`) - 最高优先级
2. **微信昵称** (`nickName`) - 备用显示
3. **默认值** ("微信用户" 或 "未设置") - 兜底显示

## 自动更新机制

### 登录成功后自动更新
1. 用户登录成功后，`app.js` 调用 `updateGlobalData()`
2. 该方法会遍历所有当前页面，调用 `onUserInfoUpdate` 方法
3. 各页面实时更新显示的用户信息

### 员工绑定成功后更新
1. 员工绑定成功后，后端返回最新的用户信息
2. 本地存储更新为最新的用户信息
3. 全局数据更新，通知所有页面刷新显示

## 测试验证

### 测试步骤
1. 启动小程序，观察个人中心页面显示
2. 登录成功后，检查是否显示真实姓名
3. 如果是新用户，绑定员工号后检查姓名是否更新
4. 切换到个人资料页面，检查所有字段显示是否正确

### 预期结果
- 登录成功后，个人中心显示真实姓名
- 员工绑定成功后，姓名立即更新
- 所有页面都能正确显示最新的用户信息

## 注意事项

1. **后端字段**: 确保后端返回的用户信息包含 `realName` 字段
2. **员工号字段**: 使用 `employeeCode` 而不是 `employeeId`
3. **实时更新**: 登录和绑定成功后会自动更新所有页面显示
4. **兜底显示**: 如果真实姓名为空，会显示昵称或默认值
