# 报告列表显示修复说明

## 问题描述

用户反馈报告列表页面显示有问题：
1. 第一部分缺少label且应该显示工作内容
2. 第二部分应该显示遇到的问题
3. 第三部分应该显示下一步计划

## 问题分析

通过查看代码发现，报告列表页面的数据格式化逻辑存在问题：
- `formatReportForComponent` 函数没有正确映射 `issues` 字段
- `list-item-card` 组件缺少对 `issues` 字段的显示支持
- 数据映射逻辑不完整

## 修复内容

### 1. 修复数据格式化逻辑

在 `pages/report/report.js` 中修改了 `formatReportForComponent` 函数：

```javascript
// 修复前
return {
  // ...
  summary: report.summary || report.content || '',
  workContent: report.workContent || report.content || '',
  nextPlan: report.nextPlan || report.plan || '',
  // ...
};

// 修复后
return {
  // ...
  // 工作内容 - 第一部分
  workContent: report.content || report.workContent || '',
  // 遇到的问题 - 第二部分  
  issues: report.issues || '',
  // 下一步计划 - 第三部分
  nextPlan: report.plan || report.nextPlan || '',
  // ...
};
```

### 2. 添加issues字段显示

在 `components/list-item-card/list-item-card.wxml` 中添加了issues部分的显示：

```xml
<!-- 遇到的问题 -->
<view class="issues-section" wx:if="{{item.issues}}">
  <view class="section-title">遇到的问题</view>
  <view class="issues-text">{{item.issues}}</view>
</view>
```

### 3. 添加CSS样式

在 `components/list-item-card/list-item-card.wxss` 中为issues部分添加了样式：

```css
/* 遇到的问题 */
.issues-section {
  margin-bottom: 16rpx;
}

.issues-text {
  background-color: #f8f9fa;
  padding: 16rpx;
  border-radius: 8rpx;
  border: 1rpx solid #e9ecef;
  color: #212529;
  line-height: 1.6;
  font-size: 26rpx;
  white-space: pre-line;
}
```

## 修复效果

修复后，报告列表页面将正确显示：

1. **第一部分**：显示"工作内容"标签和内容
2. **第二部分**：显示"遇到的问题"标签和内容  
3. **第三部分**：显示"下一步计划"标签和内容

## 数据映射关系

- `report.content` → 工作内容
- `report.issues` → 遇到的问题
- `report.plan` → 下一步计划
