# 任务列表页参数传递优化说明

## 问题描述

在任务列表页面中，需要根据不同的列表类型传递不同的参数到后端接口：
- **我的任务**：需要传递当前用户的员工ID作为`assigneeId`
- **我指派的**：需要传递当前用户的员工ID作为`assignerId`
- **指派给我的**：这个标签页不再需要，需要移除

## 问题原因分析

### 1. 原有实现的问题

**参数传递不准确：**
- 原来使用`type`参数区分列表类型
- 后端无法准确识别当前用户身份
- 可能导致数据权限问题

**标签页冗余：**
- "指派给我的"标签页功能重复
- 增加了用户界面的复杂性
- 不符合实际业务需求

### 2. 技术实现问题

**用户身份识别：**
- 需要获取当前登录用户的员工ID
- 需要调用`auth.getUserInfo()`方法
- 异步操作需要正确处理

**API调用逻辑：**
- 不同标签页需要不同的参数
- 需要重构API调用逻辑
- 避免重复的代码

## 修复方案

### 1. 移除冗余标签页

**修复思路：**
- 从`primaryTabsData`中移除"指派给我的"标签
- 保留"我的任务"和"我指派的"两个标签
- 简化用户界面


### 2. 重构参数传递逻辑

**修复思路：**
- 根据标签页类型传递不同的参数
- "我的任务"：传递`assigneeId`
- "我指派的"：传递`assignerId`
- 使用异步方式获取用户信息

**修复代码：**
```javascript
// 根据当前标签页状态添加筛选条件
if (this.data.activePrimaryTab === 'myTasks') {
  // 我的任务：传递当前用户的员工ID作为assigneeId
  this.getCurrentUserInfo().then(userInfo => {
    params.assigneeId = userInfo.employeeId;
    this.callTaskListAPI(params);
  }).catch(error => {
    console.error('获取用户信息失败:', error);
    wx.hideLoading();
    wx.showToast({
      title: '获取用户信息失败',
      icon: 'none'
    });
  });
  return; // 提前返回，避免重复调用API
} else if (this.data.activePrimaryTab === 'assignedByMe') {
  // 我指派的：传递当前用户的员工ID作为assignerId
  this.getCurrentUserInfo().then(userInfo => {
    params.assignerId = userInfo.employeeId;
    this.callTaskListAPI(params);
  }).catch(error => {
    console.error('获取用户信息失败:', error);
    wx.hideLoading();
    wx.showToast({
      title: '获取用户信息失败',
      icon: 'none'
    });
  });
  return; // 提前返回，避免重复调用API
}
```

### 2. 参数构建策略

**参数类型：**
- `assigneeId`：任务负责人ID（我的任务）
- `assignerId`：任务指派者ID（我指派的）
- `status`：任务状态筛选（二级标签页）

**参数优先级：**
- 一级标签页参数优先
- 二级标签页参数作为补充
- 分页参数始终存在

### 3. 错误处理机制
