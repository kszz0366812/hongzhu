# 员工绑定安全机制说明

## 概述

为了确保系统安全性，防止未绑定员工号的用户进行越权操作，我们实现了强制员工绑定机制。

## 安全机制

### 1. 强制绑定流程

- **检测未绑定状态**：当用户尝试执行需要员工ID的操作时，系统会自动检测用户是否已绑定员工号
- **阻止操作**：如果用户未绑定员工号，系统会：
  - 拒绝返回用户信息
  - 自动弹出员工绑定弹窗
  - 阻止任何需要员工ID的操作执行

### 2. 绑定失败重试

- **无限重试**：如果员工绑定失败，系统会：
  - 显示错误提示
  - 自动重新显示绑定弹窗
  - 不允许用户跳过绑定流程
  - 不允许用户进行任何其他操作

### 3. 技术实现

#### auth.js 中的安全检查

```javascript
// 获取用户信息（自动处理登录和绑定）
const getUserInfo = () => {
  const userInfo = wx.getStorageSync(currentConfig.STORAGE_KEYS.USER_INFO)
  
  // 如果没有用户信息，触发静默登录
  if (!userInfo) {
    // 自动触发静默登录
    return app.performGlobalSilentLogin()
  }
  
  // 如果有用户信息但没有员工ID，触发员工绑定
  if (!userInfo.employeeId) {
    // 自动触发员工绑定弹窗
    app.showGlobalEmployeeBindingModal()
    // 不返回用户信息，防止越权操作
    return Promise.reject(new Error('用户未绑定员工号，请先完成绑定'))
  }
  
  return Promise.resolve(userInfo)
}
```

#### app.js 中的绑定重试机制

```javascript
// 绑定失败后重新显示输入弹窗
.catch((error) => {
  wx.showToast({
    title: error.message || '绑定失败',
    icon: 'none'
  })
  // 绑定失败后重新显示输入弹窗
  setTimeout(() => {
    this.showGlobalEmployeeInputPrompt()
  }, 1500)
})
```

## 使用场景

### 需要员工ID的操作

以下操作需要用户已绑定员工号：

1. **提交报告**：`pages/create-report/create-report.js`
2. **创建任务**：需要员工ID的操作
3. **项目操作**：需要员工ID的操作
4. **其他需要身份验证的操作**

### 所有操作都需要员工ID

**重要**：所有用户都必须绑定员工号才能使用系统功能，包括：

1. **个人中心**：`pages/personal/personal.js`
2. **个人信息**：`pages/personal-info/personal-info.js`
3. **测试页面**：`pages/test-login/test-login.js`
4. **提交报告**：`pages/create-report/create-report.js`
5. **其他所有功能**

未绑定员工号的用户无法访问任何页面或功能。

## 安全优势

1. **防止越权**：未绑定员工号的用户无法执行任何需要身份验证的操作
2. **强制绑定**：用户必须完成员工绑定才能使用系统功能
3. **自动重试**：绑定失败时自动重试，确保绑定成功
4. **无跳过机制**：用户无法跳过或取消绑定流程

## 用户体验

- **透明处理**：用户无需手动触发登录或绑定
- **自动引导**：系统自动引导用户完成必要的步骤
- **清晰提示**：绑定失败时提供清晰的错误提示
- **无缝体验**：绑定成功后自动继续用户原本的操作

## 注意事项

1. **网络问题**：如果网络连接不稳定，绑定可能会多次重试
2. **员工号格式**：系统会验证员工号格式，格式不正确会要求重新输入
3. **服务器错误**：如果服务器返回错误，系统会显示具体错误信息并重试
4. **用户耐心**：绑定过程可能需要一些时间，用户需要耐心等待

## 测试建议

1. **测试未绑定状态**：清除本地存储，测试未绑定用户的体验
2. **测试绑定失败**：使用无效的员工号，测试失败重试机制
3. **测试网络异常**：模拟网络错误，测试错误处理
4. **测试绑定成功**：使用有效的员工号，测试成功后的流程
